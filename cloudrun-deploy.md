# Cloud Run デプロイ設定メモ

## ログイン後の表示遅延対策（最重要）

ログイン後にダッシュボードが表示されるまでの体感遅延（3〜5秒）の主因は、
Cloud Run のコールドスタートです。無アクセスが続くとインスタンスが 0 まで
スケールダウンし、次回アクセス時に起動（2〜4秒）が発生します。

これを根本的に解消するため、**常時最低1インスタンスを起動**する設定にします。

```bash
gcloud run services update kpidash-backend \
  --region asia-northeast1 \
  --min-instances 1
```

> ⚠️ 常時1インスタンスを起動するため Cloud Run の月額課金が発生します。
> 上記コマンドはインフラに対する変更のため、実行前に必ず関係者へ確認してください。

## 補足

- min-instances=1 によりコールドスタートが解消されると、インメモリキャッシュ
  （`app/services/cache_service.py`）もインスタンス再起動で消えにくくなり、
  キャッシュヒット率が安定します。
- 起動時のキャッシュウォーミング（`app/main.py` の `warm_cache()`）は
  バックグラウンドで実行され、起動自体は遅延させません。
- フロントエンドはログイン直後に主要APIを `preload()`（`frontend/src/hooks/useAuth.ts`）
  しており、ページ遷移と並行してデータ取得を開始します。
